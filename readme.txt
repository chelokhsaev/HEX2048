В процессе создания игры я решил сделать систему, в которой игровые элементы не были бы наследниками MonoBehavior, а управлялись централизованно из одного класса. Это не ECS конечно, но как в дальнейшем показали тесты - при большом количестве обновляющихся элементов скорость возрастает где то на 20%.

Main - главный класс, который в процессе игры рассылает игровым элементам апдейты, а так же содержит методы, позволяющие создать экземпляры обновляющихся игровых элементов, которые могут иметь, а могут и не иметь представления на сцене в виде transform.

Модель данных одна, контроллер по нажатию клавиш - QWASZX или свайпов на тачскрине меняет данные, изменение которых отображаются на сцене в трех разных вью. Сами игровые поля настраиваются с помощью ассетов ScriptableObject, в которых задается фабрика для создания фишек, а так же будет ли поле наклонятся. Далее эти ассеты закинуты в свойства компонента Initializer.cs у объектов на сцене. При запуске Main заменяет эти компоненты на заданные игровые элементы.

При запуске генерятся текстуры с числами для двух видов фишек. Пул для фишек и окончание игры пока не делал.
